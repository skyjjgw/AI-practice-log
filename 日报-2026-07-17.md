# 日报 - 2026-07-17

## 一、今日总结

这几天主要在整理自己后端和 AI 应用方向的学习路线，目标就是把“学什么、先学什么、最后怎么落到项目里”这件事想清楚。原来更多是看到什么学什么、刷到什么看什么，现在开始慢慢把它整理成一条比较清晰的路线。

这次整理不只是简单列了一个技术清单，更重要的是把每个阶段到底学什么、这些技术之间怎么接起来、最后能做出什么项目，慢慢理顺了。整体路线大概是从 Python、Java、Spring Boot、MySQL、Redis、Docker 这些后端基础，一路延伸到 FastAPI、LangChain、LangGraph、RAG、Agent 这些 AI 应用开发内容，方向比之前清楚很多。

## 二、核心产出与关键认知

### 1. 学习路线框架已初步成型

现在已经把整体学习过程先拆成了六个阶段，每个阶段对应的技术栈和资料方向也基本对上了如下：

- 环境搭建：Python 基础、FastAPI 入门、Java 基础、Spring Boot 入门、Git 基础与团队协作
- 数据持久化：MySQL、MyBatis-Plus、Spring Boot + MyBatis-Plus 整合、Redis 基础
- 接口规范：Spring Boot 进阶、统一响应/异常、Swagger/Knife4j、日志规范
- AI 核心能力：FastAPI 进阶、Pydantic、依赖注入、大模型 API 调用、LangChain、RAG、LangGraph、Agent
- 双剑合璧：Agent + RAG + LangChain 整合、企业级 Agent 实战、LangSmith 调试追踪
- 部署交付：Docker、Docker Compose、Streamlit 快速搭建界面与应用

这样之后学习就不是东一榔头西一棒子了，而是可以按阶段往下推，思路会更稳一些。

### 2. 技术栈之间的连接关系更加清晰

通过这次梳理，我也更明显地感觉到这些技术不是分开的，而是围着项目开发这条线串起来的：

- Python + FastAPI 负责 AI 服务接口开发与模型能力封装
- Java + Spring Boot 负责传统后端能力、企业级接口规范与业务系统理解
- MySQL / Redis 负责数据存储、缓存与项目支撑
- LangChain / LangGraph / RAG / Agent 负责 AI 应用层能力构建
- Docker / Streamlit 负责项目部署与展示交付

这也说明我现在的学习重点，已经不是单纯记知识点了，而是开始试着从“怎么把一个东西做出来”这个角度去理解整条链路。

### 3. 学习资源筛选思路更加明确

这次整理不只是把技术列出来，也顺手把看什么教程、怎么搜资料这件事一起想了：

- 尚硅谷系列：比较适合打 Java、Spring、MySQL、Redis、Docker 这些基础
- 2025 新版 FastAPI / LangChain / LangGraph / Agent / RAG 教程：更适合跟 AI 工程化这部分的新内容
- 通过“技术名 + 年份 + 场景关键词”去搜，可以更快筛掉太旧的教程

这一点也让我意识到，后面学习不能只拼时长，更要看资料选得准不准。

### 4. 项目导向意识进一步增强

从这次路线规划里也能看出来，我现在学这些内容已经不只是为了学语法，而是想慢慢服务到后面的项目里，比如：

- 后端接口开发
- AI API 调用与封装
- RAG 知识库问答系统
- Agent 工作流搭建
- 企业级 AI 应用原型
- Docker 化部署与页面展示

也就是说，后面的学习会更偏向“做项目、能展示、能汇报”这条线，方向会更聚焦一点。

## 三、遇到问题与解决方案

**问题描述：** 在整理这套学习路线之前，最大的感觉就是东西太多、太散了。基础和 AI 方向的内容混在一起，看教程的时候也容易纠结到底先学哪个，最后就很容易变成“这边看一点，那边补一点”，但整体进度并不扎实。

**解决方案：** 这次就换了一个思路，先按阶段拆，再把每个阶段要学的技术、适合看的资源、搜索关键词都整理出来。这样一来，先学基础、再进 AI、最后走到项目落地，这条线就顺了很多，原来零散的内容也慢慢变成了一张可以照着执行的学习路线图。
